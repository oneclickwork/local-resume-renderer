## 项目经历

::左右 **企业知识库 RAG 系统** || **2024.03-2024.08**

**项目简介：** 面向企业内部文档构建检索增强问答系统，支持 PDF、Markdown、Word 和网页内容入库，覆盖制度查询、产品知识与技术文档问答场景。

**主要工作：**

**1. 文档解析与切分**

- 使用 Python、FastAPI 和 PostgreSQL 构建文档处理服务，实现解析、清洗、切分、向量化与索引更新流程。
- 基于标题层级和句子边界设计结构化切分策略，保留来源页码与章节信息，降低关键信息被截断的概率。

**2. 混合检索与重排**

- 结合向量检索与 BM25 关键词检索，使用 RRF 融合结果，并通过重排模型优化 Top-K 相关性。
- 建立离线评测集，持续跟踪 Recall@5、MRR 和引用准确率，检索召回率由 **81% 提升至 93%**。

**3. 工程化与可观测性**

- 增加任务状态持久化、失败重试和结构化日志，支持批量导入过程中断恢复与问题定位。
- 使用 Playwright 完成端到端测试，覆盖文档上传、检索问答、引用溯源和权限校验等核心流程。
- 使用 Docker 编排应用、数据库和向量服务，并整理环境变量与部署脚本，降低本地开发和测试环境的配置成本。

**项目成果：** 累计处理 **2000+** 份文档，核心问答平均响应时间控制在 3 秒以内。

::左右 **智能报告生成与导出工具** || **2023.10-2024.01**

**项目简介：** 面向需要重复制作周报、项目总结和简历的场景，开发一套本地文档排版与多格式导出工具，将内容编辑和版式渲染分离。

- 设计 Markdown 扩展语法和解析流程，支持章节标题、项目符号、粗体强调、多列信息以及左右对齐的时间行。
- 使用 HTML/CSS 构建 A4 打印模板，通过统一字号、行高、页边距和分页规则保证不同文档的排版稳定。
- 接入 Playwright 输出 PDF、PNG 和 JPG，使各格式共用同一渲染页面，减少重复调整样式的工作量。
- 实现目标页数适配，根据实际溢出程度自动调整上下边距和行距，让少量超页内容无需删减即可回到一页。
- 增加字体检测、图片尺寸检查和 PDF 页数验证，避免中文乱码、内容截断以及异常空白页。
- 每次运行创建独立的日期时间目录，同时归档输入 Markdown、HTML 和全部导出文件，保留历史版本便于对比。

**项目成果：** 将原本需要反复手动调整的文档导出流程整合为一条命令，可在本地完成生成、检查和归档。
